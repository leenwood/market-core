# market-core

Backend-микросервис каталога товаров для маркетплейса. Предоставляет управление товарами и категориями, полнотекстовый поиск на базе PostgreSQL, фильтрацию, аналитику, избранное и историю поиска.

[English version](README.md)

---

## Возможности

- **CRUD товаров** — создание, получение, обновление, мягкое удаление
- **Иерархия категорий** — дерево категорий с рекурсивными запросами по подкатегориям
- **Атрибуты товаров** — бренд, цена, рейтинг, наличие, произвольные JSONB-поля
- **Полнотекстовый поиск** — `websearch_to_tsquery` по колонке `tsvector` (обновляется через триггер)
- **Нечёткий поиск** — триграммное сходство через `pg_trgm`, устойчивость к опечаткам
- **Комбинированное ранжирование** — `ts_rank_cd × 0.7 + word_similarity × 0.3`
- **Фильтрация** — диапазон цен, бренд, наличие, категория (включая подкатегории), динамические JSONB-фильтры
- **Сортировка** — по релевантности, цене, дате создания, популярности (количество просмотров)
- **Пагинация** — `page`/`page_size` на всех списочных эндпоинтах
- **Автодополнение** — префиксный поиск по названиям товаров и прошлым запросам
- **Аналитика поиска** — запись поисковых запросов, популярные запросы (за 30 дней), отслеживание кликов (CTR)
- **Избранное** — списки товаров по пользователям
- **История поиска** — история запросов по пользователям
- **Метрики Prometheus** — счётчики запросов, гистограммы задержек
- **Структурированные логи** — JSON через `log/slog`

## Быстрый старт

### Требования

- Go 1.23+
- Docker & Docker Compose
- [Task](https://taskfile.dev/#/installation) (`brew install go-task`)

### 1. Настройка окружения

```bash
cp .env.example .env
# Отредактируйте .env при необходимости (значения по умолчанию работают с docker-compose)
```

### 2. Запуск базы данных

```bash
task docker:up
```

### 3. Применение миграций

```bash
task migrate:up
```

### 4. Запуск сервера

```bash
task run
# Сервер запускается на :8080
```

## Справочник API

Базовый путь: `/api/v1`

### Товары

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/products` | Создать товар |
| `GET` | `/products` | Список товаров с фильтрацией и пагинацией |
| `GET` | `/products/{id}` | Получить товар по ID |
| `PUT` | `/products/{id}` | Обновить товар |
| `DELETE` | `/products/{id}` | Мягкое удаление товара |

**Query-параметры для списка / фильтрации:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `category_id` | UUID | Фильтр по категории |
| `include_subcategory` | bool | Включить подкатегории (по умолчанию `false`) |
| `brand` | string | Название бренда (частичное совпадение) |
| `min_price` | float | Минимальная цена |
| `max_price` | float | Максимальная цена |
| `in_stock` | bool | Наличие товара |
| `sort_by` | string | `price` / `created_at` / `popularity` / `rating` |
| `sort_dir` | string | `asc` / `desc` |
| `page` | int | Номер страницы (по умолчанию `1`) |
| `page_size` | int | Элементов на странице (по умолчанию `20`, максимум `100`) |

### Категории

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/categories` | Создать категорию |
| `GET` | `/categories` | Получить полное дерево категорий |
| `GET` | `/categories/{id}` | Получить категорию по ID |
| `DELETE` | `/categories/{id}` | Удалить категорию |

### Поиск

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/search` | Полнотекстовый поиск с фильтрами |
| `GET` | `/search/autocomplete` | Подсказки автодополнения |

**Query-параметры поиска:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `q` | string | Поисковый запрос (FTS + нечёткий поиск) |
| `category_id` | UUID | Фильтр по категории (включая подкатегории) |
| `brand` | string | Фильтр по бренду |
| `min_price` / `max_price` | float | Диапазон цен |
| `in_stock` | bool | Фильтр по наличию |
| `sort_by` | string | `relevance` / `price` / `created_at` / `popularity` |
| `sort_dir` | string | `asc` / `desc` |
| `page` / `page_size` | int | Пагинация |

Передайте заголовок `X-User-ID: <uuid>` для записи истории поиска.

### Избранное

Все запросы требуют заголовка `X-User-ID: <uuid>`.

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/favorites` | Список избранных товаров |
| `POST` | `/favorites?product_id={id}` | Добавить в избранное |
| `DELETE` | `/favorites?product_id={id}` | Убрать из избранного |

### Аналитика

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/analytics/popular-queries?limit=10` | Топ поисковых запросов (за 30 дней) |
| `GET` | `/analytics/popular-products?limit=10` | Самые просматриваемые товары |

### Системные

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/health` | Проверка работоспособности |
| `GET` | `/metrics` | Метрики Prometheus |

## Примеры сценариев

**Сценарий 1 — текстовый поиск с опечатками:**

```
GET /api/v1/search?q=iphone+15&sort_by=relevance&page=1&page_size=20
```

Сервис ищет совпадения по FTS-индексу, дополнительно применяет триграммное сходство для устойчивости к опечаткам и ранжирует результаты по формуле `ts_rank_cd × 0.7 + word_similarity × 0.3`.

**Сценарий 2 — комбинированный фильтр + текстовый поиск:**

```
GET /api/v1/search?q=pro+max&category_id=<uuid>&brand=Apple&max_price=120000&in_stock=true&sort_by=relevance
```

Полнотекстовый поиск применяется поверх SQL-фильтров в одном запросе.
